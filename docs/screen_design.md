# 화면 설계서 (v1~v1.2)

## 1. 전역 레이아웃
- 좌측 사이드바: 메뉴 그룹 2개 (경영 판단, 데이터 업로드/입력)
- 상단 바: 회사명, 기준일, 사용자 역할 표시
- 본문: 카드/차트 중심의 요약 뷰

## 2. 화면 목록
### 2.1 대표 대시보드
- 목적: 전체 위험 신호 요약
- 구성:
  - 위험 신호 카드 5개 (매출/재고/거래처/납기/재무)
  - 핵심 KPI 4개 (최근 매출, 재고회전, 연체 거래처, 납기 지연률)
  - 최근 4주 트렌드 라인 차트
  - 알림 리스트 (임계치 초과 항목)

### 2.2 매출 구조
- 목적: 매출 집중도와 구조 파악
- 구성:
  - 파이 차트: 거래처/품목/채널별 매출 비중
  - 바 차트: 상위 N 거래처 매출
  - 기간 필터 (주/월/분기)

### 2.3 재고 위험
- 목적: 과다/부족/장기재고 파악
- 구성:
  - 위험 구간 게이지 (과다/적정/부족)
  - 장기재고 테이블 (품목, 재고일수, 금액)
  - 재고회전율 추이

### 2.4 거래처 위험
- 목적: 연체/미수 및 집중 리스크
- 구성:
  - 연체 거래처 수 카드
  - 거래처별 위험등급 테이블
  - 상위 거래처 집중도 차트

### 2.5 인덴트 납기
- 목적: 납기 지연 및 리드타임 관리
- 구성:
  - 지연률 KPI
  - 지연 리드타임 분포 히스토그램
  - 지연 상위 거래처/품목 테이블

### 2.6 재무지표
- 목적: 재무제표 기반 핵심 지표 확인
- 구성:
  - 유동비율/부채비율/영업이익률 카드
  - 현금흐름 요약 카드
  - 기간 비교 (전년 대비)

### 2.7 매출 업로드
- 입력 방식: CSV/엑셀 업로드 + 간단 매핑
- 필드: 년도, 월, 일자-No., 품목코드, 품목명, 메이커, 수량, 단가, 공급가액, 코드, 사용상호(엔터인 조회화면), 통합(실제상호), 구분

### 2.8 인덴트 오더 입력
- 입력 방식: 테이블 행 추가
- 필드: 오더일, 납기일, 거래처, 품목, 수량

### 2.9 인덴트 매입 입력
- 입력 방식: 테이블 행 추가/업로드
- 필드: 매입일, 납기일, 거래처, 품목, 수량, 단가

### 2.10 거래처 관리
- 기능: 거래처 등록/수정/비활성화
- 필드: 거래처명, 사업자번호, 담당자, 연락처, 신용등급

### 2.11 재무제표 업로드
- 입력 방식: 파일 업로드 (손익계산서/대차대조표)
- 처리: 업로드 완료 후 재무지표 자동 표시

## 3. 권한/플로우
- 대표: 경영 판단 메뉴만 접근 가능
- 직원: 데이터 업로드/입력 메뉴만 접근 가능

## 4. 공통 컴포넌트
- 기간 필터, 위험 단계 뱃지, KPI 카드, 다운로드 버튼
- 테이블: 검색/정렬/필터 기본 제공
