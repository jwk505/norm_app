# 요구사항 정의서

## 1. 목적
ERP 없이도 대표가 5분 내 회사 위험 상태를 판단할 수 있는 최소 기능 시스템을 구축한다. 핵심 데이터만 시각화하여 판단 속도를 높인다.

## 2. 시스템 컨셉
- ERP 대체가 아니라 ERP 위에 얹는 "경영 판단 계기판"
- 판단(대표)과 입력(직원)의 역할 분리
- 실시간 위험 탐지 + 정기 재무 진단 구조

## 3. 범위 (v1~v1.2)
### 3.1 대표 판단 화면 (읽기 전용)
1) 대표 대시보드
2) 매출 구조 분석
3) 재고 위험 분석
4) 거래처 위험
5) 인덴트 납기 위험
6) 재무지표 대시보드 (재무제표 업로드 후)

### 3.2 데이터 입력 화면
1) 매출 입력
2) 인덴트 오더 입력
3) 인덴트 매입 입력
4) 거래처 등록/관리
5) 재무제표 업로드

## 4. 사용자/권한
- 대표: 읽기 전용, 판단 화면 접근
- 직원: 입력/업로드 화면 접근

## 5. 메뉴 구조 (IA)
### 5.1 경영 판단
- 대표 대시보드
- 매출 구조
- 재고 위험
- 거래처 위험
- 인덴트 납기
- 재무지표

### 5.2 데이터 업로드/입력
- 매출 업로드
- 인덴트 오더 입력
- 인덴트 매입 입력
- 거래처 관리
- 재무제표 업로드

## 6. 핵심 기능 요구사항
### 6.1 대표 대시보드
- 위험 요약: 매출/재고/거래처/납기/재무 위험 신호 표시
- 핵심 KPI: 최근 4주 매출, 재고회전, 연체 거래처 수, 납기 지연률
- 알림: 위험 임계치 초과 항목 하이라이트

### 6.2 매출 구조 분석
- 제품/거래처/채널별 매출 비중
- 상위 N 집중도 (예: 상위 5 거래처 매출 비중)
- 증감 트렌드 (주간/월간)

### 6.3 재고 위험 분석
- 재고 과다/부족 위험 구간 표시
- 장기재고(예: 90일 초과) 리스트
- 회전율 변화

### 6.4 거래처 위험
- 연체/미수 현황 요약
- 거래처별 신용위험 등급(간단 규칙 기반)
- 주요 거래처 집중 리스크

### 6.5 인덴트 납기 위험
- 오더 대비 납기 지연률
- 지연 리드타임 분포
- 지연 상위 품목/거래처

### 6.6 재무지표 대시보드
- 재무제표 업로드 후 핵심 지표 표시
- 유동비율, 부채비율, 영업이익률, 현금흐름 요약

### 6.7 데이터 입력 화면
- 각 입력 화면은 최소 필드 + 업로드 템플릿 제공
- 기본 검증: 필수값, 날짜 형식, 금액 음수 여부

## 7. 데이터 요구사항 (초기)
- 매출: 일자, 거래처, 품목, 수량, 금액, 채널
- 인덴트 오더: 오더일, 납기일, 거래처, 품목, 수량
- 인덴트 매입: 매입일, 납기일, 거래처, 품목, 수량, 단가
- 거래처: 거래처명, 사업자번호, 담당자, 연락처, 신용등급
- 재무제표: 손익계산서/대차대조표 업로드

## 8. 비기능 요구사항
- 5분 내 판단 가능한 요약 중심 화면
- 반응형 지원 (대표는 모바일 조회 가능)
- 데이터 입력은 최소 단계/클릭
- 로그: 업로드 실패 및 오류 기록

## 9. 제외 범위
- ERP 수준의 전표/결재/재무회계 기능
- 복잡한 권한 체계 (v1 기준)
